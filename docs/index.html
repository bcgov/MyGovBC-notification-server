A versatile notification API server

Versatile

  • Authenticated or anonymous subscriptions
  • Push and in-app pull notifications
  • Email and SMS push notification channels
  • Unicast and broadcast message types

How NotifyBC works →

Non-intrusive

  • Loose coupling: interacts with user browser or other server components through RESTful API.
  • Handles common backend business logic only. Impose no frontend UI so you can design UI widgets and workflow native to your web site to capture user subscriptions or display in-app notifications.

Explore the API →

Microservice

Containerized deployment to PAAS such as OpenShift. No need for complex enterprise-wide shared service. Better meet privacy mandates.

Deployment Methods →

Get up and running in seconds.

Quick-start Instructions

~ $ git clone https://github.com/bcgov/MyGovBC-notification-server.git notifyBC

~ $ cd notifyBC

~/notifyBC $ npm install

~/notifyBC $ npm start

# => Now browse to http://localhost:3000/explorer
